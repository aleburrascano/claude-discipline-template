# CLAUDE.md map

Index of every `CLAUDE.md` file in the repo and what each owns. Use to answer "is there a CLAUDE.md for X?" without searching.

## Hierarchy

| Path | Scope | Owns |
|---|---|---|
| `~/.claude/CLAUDE.md` (global, not in repo) | User-wide | Universal discipline: Karpathy 4 principles, sacred-tests, verification, cited claims, brevity, knowledge sources, token discipline |
| `CLAUDE.md` (this repo's root) | Project | Project summary, hexagonal/vertical-slice rules, AIDEV anchor convention, skill auto-activation pointer, commit discipline, nested-context pointer |
| `<your-mobile-or-frontend>/CLAUDE.md` | Mobile global | Expo/React Native quirks, navigation, theming, performance defaults |
| `<your-mobile-or-frontend>/src/features/_template/CLAUDE.md` | Template | The shape every feature CLAUDE.md should follow when generated |
| `<your-mobile-or-frontend>/src/features/<feat>/CLAUDE.md` | Feature-local | Feature glossary (terms specific to this feature), patterns specific here, known gotchas. Auto-maintained block regenerated by `/update-nested-claude-md`. |
| `<your-backend>/CLAUDE.md` | Backend global | Python/FastAPI quirks, async, Pydantic v2, hexagonal reminders |
| `<your-backend>/src/<project-slug>/domain/CLAUDE.md` | Domain layer | Purity rule restated, DDD tactical reminders |
| `<your-backend>/src/<project-slug>/domain/<context>/CLAUDE.md` | Context-local | Bounded context's invariants, aggregates, ubiquitous-language pointer. Optional — only when context has substantial domain code. |

## Generation

- **Root + apps/mobile + services/api** — hand-maintained. Update when project-wide patterns change.
- **Feature/context nested files** — initially copied from `_template/`, then their `AUTO-MAINTAINED` blocks regenerated by the `/update-nested-claude-md` skill after every 3rd commit affecting the folder.

## Discipline

- Each CLAUDE.md has one job. The root is **constitution**, mobile-global is **platform quirks**, feature-local is **what's specific here**.
- Never repeat content. If a rule belongs in the root, don't restate it in a feature file.
- Bloat targets:
  - root CLAUDE.md: ≤60 lines
  - apps/mobile or services/api global: ≤80 lines
  - feature/context-local: ≤30 lines
  - Any CLAUDE.md > 200 lines → split (Boris's ceiling; Claude starts ignoring beyond that).

The `/audit-docs` skill checks bloat as part of its sweep.

## What goes where (decision tree)

- Rule applies to **all** files in the repo? → root `CLAUDE.md`.
- Rule applies to a **language/stack** (all TS, all Python)? → `.claude/rules/<lang>.md` with `paths:`.
- Rule applies to a **layer** (domain, application, adapters)? → `.claude/rules/<layer>-layer.md` with `paths:`.
- Rule applies to **one feature**? → that feature's nested `CLAUDE.md`.
- Rule applies to **one context**? → that context's nested `CLAUDE.md` (or skip if the layer rule already covers it).
