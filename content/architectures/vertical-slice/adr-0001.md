# ADR-0001: Vertical-Slice Architecture

- **Status:** Accepted
- **Date:** {{DATE}}
- **Deciders:** {{DECIDERS}}
- **Context tags:** [arch, foundational]

## Context

{{PROJECT_NAME}} organizes code by **feature**, not by technical layer. Each feature folder owns everything needed to deliver that feature end-to-end: routes, business logic, data access, types, tests.

This contrasts with layered (where you'd have separate `controllers/`, `services/`, `repositories/` folders all containing pieces of every feature). Vertical-slice trades global structure for per-feature cohesion.

## Decision

Layout:

```
<your-pkg>/
├── features/
│   ├── <feature-1>/
│   │   ├── routes.ts / .py
│   │   ├── handler.ts / .py
│   │   ├── types.ts / .py
│   │   ├── persistence.ts / .py    (or queries.sql)
│   │   └── __tests__/
│   ├── <feature-2>/
│   │   └── ...
│   └── _template/
└── shared/
    ├── ui/        (frontend: design system)
    ├── db/        (backend: connection pool, query runner)
    └── lib/       (utilities — must have 2+ consumers)
```

Rules:
- **Cross-feature imports are forbidden.** Features are independent units.
- **Promote to `shared/` only after 2+ real consumers** (YAGNI guard).
- **Each feature's tests live inside the feature.** Cross-feature integration tests in `tests/integration/`.

## Alternatives considered

| Alternative | Why not |
|---|---|
| **Layered** | Higher cross-feature coupling; touching one feature requires editing multiple top-level folders. |
| **Hexagonal** | More ceremony than vertical-slice needs for CRUD-heavy apps. Combinable: vertical-slice features can internally use a hexagonal layering when needed. |

## Consequences

### What becomes easier
- Adding a new feature is mechanical: copy `_template/`, rename, fill in.
- Deleting a feature is one folder removal.
- Onboarding to a feature is reading one folder.

### What becomes harder
- Cross-feature concerns (auth, observability) need to live in `shared/` from day 1 or get awkwardly duplicated.
- Discipline required to avoid cross-feature shortcuts.
